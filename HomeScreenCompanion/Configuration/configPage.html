tune Edit system prompt expand_more

The system prompt is a hidden instruction sent to the AI before every request. It tells the model what role to play and — critically — that it must respond with a specific JSON format that the plugin can parse.

This is different from what you write in a source's prompt (e.g. "Best sci-fi movies from the 90s"). That text is your actual question. The system prompt is the ruleset the AI must follow when answering it.

Only edit this if you know what you're doing. If the JSON format instruction is removed or changed, the plugin will not be able to read the AI's response and the list will appear empty.

Home Screen Companion — Setup Guide

All features at a glance
What does this plugin do?
Home Screen Companion keeps your Emby home screen up to date automatically. You connect it to movie lists (from sites like Trakt or MDBList, or your own filters), and the plugin marks those movies with a tag in your Emby library. Those tags can then power custom home screen rows — like "Trending Now" or "Top 10 Action Movies" — that update themselves on a schedule.

rocket_launch Quick Start
1
Enter API Keys — go to the Settings tab. An API key is a code that lets the plugin access a service on your behalf — think of it as a password for the plugin, not for you personally. Only add keys for the services you plan to use.
Trakt: Go to trakt.tv/oauth/applications and sign in. Click New Application, give it any name, and in the Redirect URI field type exactly: http://localhost (this is just a required placeholder — don't visit that address). Click Save, then copy the Client ID and paste it here.
MDBList: Go to mdblist.com/preferences, scroll down to the API Access section, and copy your API key.
TMDb: Go to themoviedb.org/settings/api, scroll to API Read Access Token, and copy the long token — not the short API Key above it. Public TMDb lists work without a token.
OpenAI / Gemini: Only needed if you want to use the AI created lists source type. API key fields are hidden by default — tick the Show checkbox next to each field to see what you've entered.
2
Add a Source — open the Tag & Collection tab and click + Add New Source. A source is simply the place the plugin fetches its movie list from.
3
Configure the source — each source has several tabs:
  • Source — choose where to get the list (Trakt URL, MDBList URL, local collection, etc.)
  • Tag — set the tag name that will be applied to matching movies in Emby
  • Collection — optionally also group the movies into an Emby collection
  • Schedule — optionally limit when this source is active (e.g. only in December)
4
Save & Run — click Save Settings at the bottom of the page, then click the Run button at the bottom and choose Full sync. The plugin will now fetch the lists, match movies in your library, and apply the tags. Done!
input Source Types
A source is where the plugin gets its list of movies or shows. Each source is linked to a tag — once the plugin has fetched the list, it marks matching titles in your Emby library with that tag. Choose the source type that suits how you want to build your list:
public External
Paste a list URL from Trakt, MDBList, or TMDb. The plugin downloads the list and tags every matching title it finds in your library. You can add multiple URLs to the same source — they are combined into one list.
TMDb can be used in two ways:
  • Your own lists — paste the URL to any public or private list you have created on themoviedb.org.
  • API endpoints — paste a direct TMDb API URL (e.g. popular movies, top-rated, now-playing, or a specific collection). Requires a TMDb API Read Access Token in the Settings tab.
video_library Local
Use an existing playlist or collection already in Emby as the source. Every item in that list gets the configured tag. You can also use this to show a collection as a home screen row — see Home Section — Collections mode below.
tune MediaInfo
Build a list based on the properties of your own files — for example "all 4K HDR movies" or "Action films I haven't watched yet". Filter by video quality, audio format, genre, actors, ratings, and more.
auto_awesome AI created lists
Describe what you want in plain language — for example "Best sci-fi films from the 90s" — and let OpenAI or Google Gemini generate the list. You can also have the AI take into account what a specific user has watched recently, to personalise the suggestions. Requires an API key in Settings.

local_offer Tags & Collections
What is a tag? — A tag is an invisible label that Emby can attach to a movie or show. You won't see it while browsing normally, but the plugin uses it to know which items belong to which list. Tags are what makes home screen rows work — a row simply shows all items that carry a certain tag.
label
Tag — The plugin applies a tag to every item on the list. When an item drops off the list (e.g. the Trakt list is updated), the tag is removed automatically on the next sync.
collections_bookmark
Collection — In addition to (or instead of) tagging, the plugin can maintain an Emby collection. Items are added and removed automatically on every sync, just like tags.
check_box_outline_blank
Only Collection — Enable this to skip tagging entirely and only maintain the collection. Useful when you just want a tidy collection without adding any labels to individual movies.
image
Collection Poster — Upload a local image or paste an image URL to set a custom poster for the collection. The poster is stored on the server and re-applied on every sync so it is never overwritten by Emby.
tv
Home Section per source — Each source can have its own home screen row for selected users. Set this up in the Home Screen tab inside each source card. This works independently from the Home Screen copy feature.
collections
Home Section — Collections mode — Want to show a whole collection as a row (rather than individual movies)? Use a Local collection source, set a tag name, and in the Home Screen tab choose Media Type: Collections. The plugin will tag the collection itself — not the movies inside it — so Emby shows it as a collection row. The individual movies inside the collection are never tagged in this mode.
block
Blacklist — IMDB IDs in the Blacklist tab are permanently excluded from this source, even if they appear in the fetched list.
calendar_month Scheduling
By default a source is always active. Use the Schedule tab to limit when a source runs — for example to show a Halloween horror list only in October, or a Christmas movie row only in December.
all_inclusive
No schedule — The source is always active (default).
date_range
Specific dates — The source is only active between a chosen start date and end date. Outside those dates the tag and collection are removed automatically.
repeat
Recurring — Active on specific days every week, month, or year. You can stack multiple intervals on the same source — for example "every Friday" and "every Saturday".
workspace_premium
Priority Override — When this source is in-schedule, it takes over from any other sources that share the same tag. The other sources are temporarily suppressed. Example: a "Spotlight" source runs all year, but a "Spotlight Halloween" source with Priority Override takes over during October — the Halloween list replaces the regular one automatically.
analytics MediaInfo Filters
MediaInfo lets you build a list based on the properties of your own files — think of it as building a smart search. For example: "all 4K Dolby Vision movies in the Action genre that I haven't watched yet."
How rules work:
  • Each rule checks one property against a value (e.g. Genre = Action).
  • Rules in the same group can be combined with AND (must match all) or OR (match any one).
  • Multiple groups can also be combined with AND / OR using the connector between them.
  • The NOT button on a rule inverts it — e.g. NOT Genre: Comedy matches everything except comedies.
  • For text fields (Genre, Actor, Title…), enter one value per line. Any matching line triggers the rule.
  • Text fields offer Contains (partial match) or Exact (full match) mode.
  • Number fields (Year, Rating, Runtime…) support = > ≥ < ≤.
Video
Resolution (4K, 1080p, 720p, SD, 8K), Video codec (HEVC/H.265, AV1, H.264), HDR type (HDR10, Dolby Vision, HLG)
Audio
Format (Atmos, DTS-HD MA, TrueHD, DTS, AC3, AAC), Channels (7.1, 5.1, Stereo, Mono), Audio language — one language code per line (e.g. swe, eng)
Content
Genre, Actor/Cast, Director, Writer, Studio, Title, Episode Title, Plot overview, Tag (existing Emby tag), Content rating (PG-13, R…), IMDB ID, Collection, Playlist — one value per line; any match = rule matches
Music
Artist, Album. Use MediaType: Audio to target songs or MediaType: MusicVideo for music videos. Also supports Bit Rate, Sample Rate, Bits Per Sample, Track Number, Disc Number with numeric operators.
Metrics
Community Rating, Release Year, Runtime (minutes), Date Added (days ago), Date Modified (days ago), File Size (MB) — all support numeric operators
Activity
Watched / Unwatched, Last Played (days ago), Play Count, Watched By Count (how many users have finished it) — can be scoped to any user, all users, or a specific user
home Home Screen
The Home Screen tab has two independent sub-sections: Manage (source-based rows) and Copy & Sync (copy one user's layout to others).
view_list
Manage — Creates and keeps custom home screen rows up to date. Each row is tied to one of your sources and is automatically refreshed for the selected users every time sync runs. To set up a row for a source, open that source card and go to its Home Screen tab.
copy_all
Copy & Sync — Copies the complete home screen layout from one user to one or more other users. Handy when you want all profiles to look the same. This runs separately from the tag and collection sync.
sync
Automatic & Manual — Both features run as part of the scheduled task, or you can trigger them at any time using the Run button → Sync home screen only.
format_list_numbered Top-List
A top-list is a ranked movie row on the home screen — like "My Top 10 Movies" — where each poster shows a numbered badge (1, 2, 3…) indicating its position. The plugin creates a separate Emby library containing shortcuts to the ranked movies (the actual files are not copied or moved), and shows that library as a dedicated home screen section. Found in the Top-Lists tab.
create_new_folder
Create top-list from a tag — Pick a tag managed by the plugin (movies only) and click Create top-list. The plugin builds the ranked library and sets up a home screen section for the users you choose. The order follows the original list order from the source (e.g. the order on a Trakt list). The list updates automatically whenever the plugin syncs that tag.
edit_note
Manual top-list — Click Create manual top-list to hand-pick and order movies yourself, without needing an existing tag. Search your library, drag items into the order you want, and save.
block
Automatic exclusions — Movies in a top-list are automatically hidden from all other tag-based home screen rows, so the same movie (with a number badge) never shows up in another row.
build
Top-list section sync — If a home screen row goes out of sync after adding a new Emby library, run the Top-list section sync task (via the Run button or Emby's scheduled tasks page) to rebuild all the exclusion settings.
